Internet Protocol (IP)

Das Internet Protocol (IP) ist das zentrale Kommunikationsprotokoll des Internets. Es ermöglicht die eindeutige Identifizierung und Adressierung von Geräten in Netzwerken. IP arbeitet auf der Vermittlungsschicht (Layer 3) des OSI-Modells und bestimmt, wie Datenpakete geroutet werden.

IPv4 und IPv6​

IPv4​

IPv4 ist die am weitesten verbreitete Version von IP und verwendet 32-Bit-Adressen, die in vier Dezimalzahlen (0-255) geschrieben werden, z. B. 192.168.1.1. Es gibt ca. 4,3 Milliarden mögliche IPv4-Adressen.

IPv6​

IPv6 wurde eingeführt, um die Adressknappheit von IPv4 zu beheben. Es nutzt 128-Bit-Adressen, geschrieben in hexadezimaler Form, z. B. 2001:0db8:85a3::8a2e:0370:7334. Dies ermöglicht 2^128 mögliche Adressen.

IPv6-Adressen kürzen​

IPv6-Adressen können gemäß folgender Regeln gekürzt werden:

  1. Führende Nullen in jedem Segment können weggelassen werden (2001:0db8:0000:0000:0000:ff00:0042:8329 → 2001:db8:0:0:0:ff00:42:8329).
  2. Längste zusammenhängende :0:-Blöcke können durch :: ersetzt werden (nur einmal pro Adresse erlaubt):
    2001:db8:0:0:0:ff00:42:8329 → 2001:db8::ff00:42:8329.

Adressierungstypen​

  • Unicast: Eine eindeutige Adresse für eine einzelne Netzwerkverbindung.
  • Broadcast (nur IPv4): Pakete werden an alle Geräte im Netzwerk gesendet (255.255.255.255).
  • Multicast: Pakete werden an eine Gruppe von Geräten gesendet (224.0.0.0 - 239.255.255.255 für IPv4, FF00::/8 für IPv6).
  • Anycast (nur IPv6): Mehrere Geräte teilen sich eine Adresse, wobei das nächstgelegene Gerät antwortet.

IPv4-Adressklassen​

IPv4-Adressen sind in verschiedene Klassen unterteilt:

KlasseNetzwerkteilHostteilBeispielSubnetzmaske
A8 Bit24 Bit10.0.0.0255.0.0.0
B16 Bit16 Bit172.16.0.0255.255.0.0
C24 Bit8 Bit192.168.1.0255.255.255.0

Klassen D und E sind für Multicast bzw. experimentelle Zwecke reserviert.

IPv4-Subnetting​

Subnetting unterteilt Netzwerke in kleinere Subnetze, um die IP-Adressverwaltung zu optimieren.

Beispiel:
Ein Unternehmen benötigt 16 Subnetze in einem 192.168.1.0/24-Netz.
Die Subnetzmaske wird auf /28 gesetzt:

  • Subnetz 1: 192.168.1.0/28 (14 Hosts, da 2 für Netz- und Broadcastadresse reserviert sind).
  • Subnetz 2: 192.168.1.16/28
  • … bis 192.168.1.240/28.
